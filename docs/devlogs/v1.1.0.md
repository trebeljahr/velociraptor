# Raptor Runner v1.1.0 — A New Look

Two big things happened in this cycle. The UI stopped looking like a web app, and the whole menu system got rewired onto React under the hood.

## A new visual language

Up until v1.0.4 the menus all wore the quiet uniform of a modern web app — frosted-glass pills, system fonts, soft drop shadows, rounded 12-pixel cards. Clean, fine, fundamentally anonymous. Looking at it next to the raptor running across a hand-drawn desert felt like two different games bolted together.

v1.1.0 is a full sweep. Every chrome surface now reads the same sand-on-dune language:

- **One display face.** Unbounded throughout — chunky, geometric, a little toyetic. Self-hosted so Electron and Capacitor builds work offline.
- **A palette pulled from the canvas.** Sand and sky-teal and dune-dark and coin-gold and cactus-green were already painted onto the game world; now they run the menus too. The sand panel + 4px dune border became the base vocabulary; every button, every collapsible, every badge is cut from it.
- **No glassmorphism, no backdrop-blur, no soft shadows, no pill radii.** The top-right icon cluster went from frosted circles to stamped squares. The Shop button turned into a flat sand panel with a gold coin chip. The game-over "Share your score" button stopped being an animated shimmer gradient around a black pill and became a single solid button the size of its label.
- **Decisive feedback on press.** Hovering a button does a colour swap to sky-teal. Pressing nudges the whole thing two pixels down — physical-key idiom instead of the "card lifts off the page" web-app instinct.
- **A new title plate.** "Game Over" still pops over the canvas, but now wearing a chunky dune outline instead of a blurry white halo.

If you've played v1.0.4 the first thing you'll notice is the Start screen — the dark-brown pill is gone, the title is bigger, and the "Made with ♥" footer finally sits where a footer should sit. From there every menu you open wears the same skin.

## Under the hood: the React migration

Invisible to players but a big deal for whoever maintains this next. The menu / shop / achievements / game-over / credits / start-screen chrome got ported from hand-written DOM manipulation in a 2,500-line `ui.ts` file to React components under `src/ui/react/`.

That's:

- **One component per self-contained surface**: `<Shop>`, `<Achievements>`, `<Menu>` and its subgroups (`<SoundSettings>`, `<CosmeticsMenu>`, `<MenuList>`, `<DebugSettings>`), `<ScoreCardActions>`, `<StartScreen>`, `<Credits>`, `<IframeOverlay>`.
- **Ten migration commits** landed in sequence, each pixel-identical to what it replaced — the visual redesign was a separate pass on top, so regressions from architecture and regressions from design never had to be untangled from the same change.
- **The game loop stayed vanilla.** The canvas, the RAF tick, `window.Game`'s public API, and the HUD elements that sync per-frame (score ticker, coin counter, achievement toast) are untouched. React only handles the component-tree UIs that benefit from it.

The CSS is still in `legacy.css` for now — Tailwind 4 is wired up and the `@theme` block defines the palette + display font, but utilities aren't applied inline yet. That's a future pass; the legacy file got ~650 lines lighter just by letting React own the structural DOM it used to spell out.

## Multi-target still works

Every change was re-verified against the three build targets that matter:
- **Web PWA** — service worker picks up the new bundle + the Unbounded .woff2 subsets transparently.
- **Electron desktop** — relative base paths still resolve, CSP unchanged (Tailwind's static CSS emission stays under `style-src 'self' 'unsafe-inline'`).
- **Capacitor mobile** — build:mobile clean, haptics + status bar still init through the existing bridge.

## Known stuff

- About / Imprint pages still load in iframes pointing at the standalone `about.html` / `imprint.html` files — those pages still wear the old type, since they also serve as standalone legal docs at their own URLs. They'll get the treatment in a later pass.
- No new gameplay in this cycle; v1.1.0 is a look-and-feel release.

Next up: I'll probably fold the inline Tailwind utilities into the React components screen by screen so `legacy.css` can keep shrinking. Otherwise it's back to gameplay features — the ship list was starting to feel short on 'new things to do'.
