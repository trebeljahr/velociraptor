# Raptor Runner v1.0.1 — Desktop splash screen unstuck

First-day hotfix. If you grabbed the Mac / Windows / Linux desktop build on launch day and saw the game boot to its splash screen and just… sit there — this is the release that fixes it.

## What was broken

Packaged Electron loads the app via the `file://` protocol. Vite was emitting absolute paths like `/assets/main-XXX.js`, which under `file://` resolve to the filesystem root instead of the bundle directory. The main bundle 404'd silently; the UI kept retrying the "game ready" handshake forever; the splash never faded.

The browser / PWA build was unaffected because absolute paths are fine under HTTP.

## What I did

Added a dedicated `VITE_TARGET=electron` build target that:

- Uses `base: "./"` so asset paths become relative, correctly resolving under `file://`.
- Skips the PWA service-worker plugin — service workers don't register under `file://` anyway, and having a broken SW entry in the HTML was adding noise.

Same pattern as the Capacitor mobile build (also `file://`-adjacent), just a second target alongside it.

## Action for you

If you already downloaded v1.0.0 from itch and bounced off a blank splash, grab v1.0.1. Browser players don't need to do anything — the online build was never affected.
